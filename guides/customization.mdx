## Overview

Shizen UI components are designed to be highly customizable while maintaining consistency and accessibility. This guide covers various approaches to customizing component appearance and behavior.

## Using the className Prop

Every Shizen UI component accepts a `class` prop that allows you to add custom CSS classes. These classes are merged with the component's default styles using the `cn()` utility.

```svelte
<script>
  import { Button } from '@shizen-ui/svelte';
</script>

<Button class="w-full shadow-lg">
  Custom Styled Button
</Button>
```

<Tip>
The `cn()` utility intelligently merges Tailwind CSS classes, ensuring that your custom classes take precedence when conflicts occur.
</Tip>

## Customizing with Variants

Many components support built-in variants that control their appearance. The variant system uses [tailwind-variants](https://www.tailwind-variants.org/) under the hood.

### Button Variants

Buttons come with multiple variant options:

```svelte
<script>
  import { Button } from '@shizen-ui/svelte';
</script>

<div class="flex gap-4">
  <Button variant="primary">Primary</Button>
  <Button variant="secondary">Secondary</Button>
  <Button variant="tertiary">Tertiary</Button>
  <Button variant="outline">Outline</Button>
  <Button variant="ghost">Ghost</Button>
  <Button variant="danger">Danger</Button>
  <Button variant="soft-danger">Soft Danger</Button>
</div>
```

### Size Variants

Control component sizing with the `size` prop:

```svelte
<script>
  import { Button, Input } from '@shizen-ui/svelte';
</script>

<!-- Button sizes -->
<div class="flex gap-2 items-center">
  <Button size="sm">Small</Button>
  <Button size="md">Medium</Button>
  <Button size="lg">Large</Button>
</div>

<!-- Input sizes -->
<div class="flex flex-col gap-2">
  <Input size="sm" placeholder="Small input" />
  <Input size="md" placeholder="Medium input" />
  <Input size="lg" placeholder="Large input" />
</div>
```

## Overriding Component Styles

You can override component styles at different levels of specificity:

### Component-Level Customization

Add custom classes directly to components:

```svelte
<script>
  import { Button } from '@shizen-ui/svelte';
</script>

<Button 
  class="bg-gradient-to-r from-purple-500 to-pink-500 hover:from-purple-600 hover:to-pink-600"
>
  Gradient Button
</Button>
```

### Sub-Component Customization

Compound components allow you to customize individual parts:

```svelte
<script>
  import { Checkbox, Label } from '@shizen-ui/svelte';
</script>

<Checkbox>
  <Checkbox.Control class="border-2 border-purple-500" />
  <Checkbox.Content>
    <Label class="font-bold text-purple-700">Custom Checkbox</Label>
  </Checkbox.Content>
</Checkbox>
```

### State-Based Styling

Components expose data attributes for state-based styling:

```svelte
<script>
  import { Input } from '@shizen-ui/svelte';
</script>

<style>
  /* Target invalid state */
  :global([data-invalid="true"]) {
    @apply ring-2 ring-red-500;
  }
  
  /* Target disabled state */
  :global([data-disabled="true"]) {
    @apply opacity-50 cursor-not-allowed;
  }
</style>

<Input invalid={true} />
```

## Creating Reusable Style Patterns

For consistent customization across your application, create wrapper components:

<Steps>

<Step title="Create a custom Button component">

Create a file `src/lib/components/CustomButton.svelte`:

```svelte
<script lang="ts">
  import { Button } from '@shizen-ui/svelte';
  import type { Snippet } from 'svelte';
  
  interface Props {
    variant?: 'primary' | 'secondary' | 'danger';
    size?: 'sm' | 'md' | 'lg';
    children: Snippet;
    class?: string;
  }
  
  let { variant = 'primary', size = 'md', children, class: className }: Props = $props();
</script>

<Button 
  {variant} 
  {size}
  class="font-semibold tracking-wide {className}"
>
  {@render children()}
</Button>
```

</Step>

<Step title="Use your custom component">

```svelte
<script>
  import CustomButton from '$lib/components/CustomButton.svelte';
</script>

<CustomButton variant="primary">
  Submit Form
</CustomButton>
```

</Step>

</Steps>

## Working with Style Variants

Shizen UI exposes the style variant functions for advanced customization:

```svelte
<script>
  import { buttonStyles } from '@shizen-ui/styles';
  import { cn } from '@shizen-ui/styles';
  
  // Generate custom button classes
  const customButtonClass = cn(
    buttonStyles({ variant: 'primary', size: 'lg' }),
    'rounded-full px-8'
  );
</script>

<button class={customButtonClass}>
  Custom Button
</button>
```

<Note>
The `buttonStyles()`, `inputStyles()`, and other variant functions are available from `@shizen-ui/styles` package. They return the appropriate CSS classes based on the variants you specify.
</Note>

## Theming with CSS Variables

Shizen UI uses CSS custom properties for theming. Override these variables to customize the entire design system:

```css
:root {
  /* Primary colors */
  --color-primary: 240 100% 50%;
  --color-primary-hover: 240 100% 45%;
  
  /* Border radius */
  --radius-sm: 0.25rem;
  --radius-md: 0.375rem;
  --radius-lg: 0.5rem;
  
  /* Spacing */
  --spacing-xs: 0.25rem;
  --spacing-sm: 0.5rem;
  --spacing-md: 1rem;
  --spacing-lg: 1.5rem;
}
```

## Full Width Components

Many container components support the `fullWidth` prop:

```svelte
<script>
  import { TextField, Label, Input } from '@shizen-ui/svelte';
</script>

<TextField fullWidth>
  <Label>Email Address</Label>
  <Input type="email" placeholder="you@example.com" />
</TextField>
```

## Best Practices

<Warning>
Avoid using `!important` in your custom styles. The component architecture is designed to respect custom classes without needing to override specificity.
</Warning>

- **Use the variant system first**: Before adding custom classes, check if a built-in variant meets your needs
- **Leverage composition**: Create wrapper components for commonly used style patterns
- **Maintain consistency**: Define design tokens (colors, spacing, typography) in a central location
- **Test states**: Always test your customizations in different states (disabled, invalid, focused)
- **Use data attributes**: Target component states using `data-*` attributes for more maintainable styles

### Custom Styled Checkbox

```svelte
<script>
  import { Checkbox, Label, Description } from '@shizen-ui/svelte';
  
  let checked = $state(false);
</script>

<Checkbox bind:checked class="p-4 rounded-lg border hover:bg-gray-50">
  <Checkbox.Control class="border-2 border-blue-500" />
  <Checkbox.Content class="ml-3">
    <Label class="text-base font-medium">Enable notifications</Label>
    <Description class="text-sm">Receive updates about your account</Description>
  </Checkbox.Content>
</Checkbox>
```
