## Overview

The Radio component is a selection control that allows users to choose a single option from a set. Built with a compound component pattern, it provides fine-grained control over rendering with separate Control, Indicator, and Content sub-components. Radio buttons are typically used within a RadioGroup for coordinated single-selection behavior.

## Installation

```bash
npm install @shizen-ui/svelte
```

## Compound Component Structure

The Radio uses a compound component pattern with three main parts:

- **Radio** (Root) - The main container that manages state and context
- **Radio.Control** - The visual radio button circle container
- **Radio.Indicator** - The dot indicator that appears when selected
- **Radio.Content** - The label content next to the radio button

```svelte
<Radio value="option">
  <Radio.Control>
    <Radio.Indicator />
  </Radio.Control>
  <Radio.Content>
    Label text
  </Radio.Content>
</Radio>
```

<Note>
  If no children are provided to `Radio.Control`, it will automatically render a `Radio.Indicator` as the default content.
</Note>

## Props

### Radio (Root)

<ParamField path="value" type="string" required>
  The value associated with this radio button. This is required and identifies the option within a RadioGroup.
</ParamField>

<ParamField path="checked" type="boolean" default={false}>
  The checked state of the radio button. Use `bind:checked` for two-way binding.
  
  When inside a RadioGroup, this is automatically managed by the group.
</ParamField>

<ParamField path="disabled" type="boolean" default={false}>
  When true, disables the radio button interaction. Inherited from RadioGroup or Field context if present.
</ParamField>

<ParamField path="invalid" type="boolean" default={false}>
  When true, applies invalid/error styling. Inherited from RadioGroup or Field context if present.
</ParamField>

<ParamField path="name" type="string">
  The name attribute for the underlying input element. Automatically set by RadioGroup if present.
</ParamField>

<ParamField path="id" type="string">
  The ID for the radio input. Auto-generated if not provided.
</ParamField>

<ParamField path="class" type="string">
  Additional CSS classes to apply to the root container.
</ParamField>

### Radio.Control

<ParamField path="class" type="string">
  Additional CSS classes to apply to the control circle.
</ParamField>

<ParamField path="children" type="Snippet">
  Custom content for the control. If not provided, defaults to `<Radio.Indicator />`.
</ParamField>

### Radio.Indicator

<ParamField path="class" type="string">
  Additional CSS classes to apply to the indicator dot.
</ParamField>

<ParamField path="children" type="Snippet">
  Custom indicator content. If not provided, shows a default circular dot.
</ParamField>

### Radio.Content

<ParamField path="class" type="string">
  Additional CSS classes to apply to the content wrapper.
</ParamField>

<ParamField path="children" type="Snippet" required>
  The label content to display.
</ParamField>

## Checked State Management

### Standalone Radio

```svelte
<script>
  import { Radio } from '@shizen-ui/svelte';
  
  let selected = $state(false);
</script>

<Radio value="accept" bind:checked={selected}>
  <Radio.Control />
  <Radio.Content>
    I accept the terms
  </Radio.Content>
</Radio>

<p>Selected: {selected}</p>
```

### Manual Radio Group (Not Recommended)

```svelte
<script>
  import { Radio } from '@shizen-ui/svelte';
  
  let selected = $state('option1');
</script>

<Radio value="option1" checked={selected === 'option1'} onclick={() => selected = 'option1'}>
  <Radio.Control />
  <Radio.Content>Option 1</Radio.Content>
</Radio>

<Radio value="option2" checked={selected === 'option2'} onclick={() => selected = 'option2'}>
  <Radio.Control />
  <Radio.Content>Option 2</Radio.Content>
</Radio>

<p>Selected: {selected}</p>
```

<Note>
  For multiple radio buttons that form a group, use the RadioGroup component instead of managing the state manually. RadioGroup handles selection coordination automatically.
</Note>

## Accessibility

- Uses a native `<input type="radio">` element for proper semantics
- Supports keyboard navigation:
  - **Space** or **Enter** to select
  - **Tab** to focus the checked radio or first radio in group
  - **Arrow keys** for navigation within a RadioGroup
- Properly handles `aria-checked` attribute
- Keyboard focus management:
  - Checked radio has `tabindex="0"`
  - Unchecked radios in a group have `tabindex="-1"` 
  - Standalone radio has `tabindex="0"`
- Disabled radios have `tabindex="-1"` to remove from keyboard navigation
- The entire radio container is clickable, improving usability
- Data attributes (`data-checked`, `data-disabled`, `data-invalid`) are applied for styling

```svelte
<Radio value="option" aria-label="Choose this option">
  <Radio.Control />
  <Radio.Content>
    Option label
  </Radio.Content>
</Radio>
```

## Usage with RadioGroup

When a Radio is placed inside a RadioGroup, it automatically coordinates with the group:

```svelte
<script>
  import { Radio, RadioGroup } from '@shizen-ui/svelte';
  
  let selected = $state('option1');
</script>

<RadioGroup bind:value={selected}>
  <Radio value="option1">
    <Radio.Control />
    <Radio.Content>Option 1</Radio.Content>
  </Radio>
  
  <Radio value="option2">
    <Radio.Control />
    <Radio.Content>Option 2</Radio.Content>
  </Radio>
  
  <Radio value="option3">
    <Radio.Control />
    <Radio.Content>Option 3</Radio.Content>
  </Radio>
</RadioGroup>

<p>Selected: {selected}</p>
```

<Tip>
  The Radio component automatically detects when it's inside a RadioGroup and coordinates its checked state with the group's value, eliminating the need for manual state management on each radio button.
</Tip>

## Context Propagation

The Radio root component provides context to its sub-components:

- `checked` - Whether this radio is selected
- `disabled` - Whether the radio is disabled
- `invalid` - Whether the radio has validation errors
- `id` - The radio input ID

Sub-components automatically receive this context and don't need props passed explicitly.

## Data Attributes

The following data attributes are applied for styling:

- `data-checked` - Applied when the radio is selected
- `data-disabled` - Applied when disabled
- `data-invalid` - Applied when invalid
- `data-custom` - Applied to Indicator when using custom content

<Note>
  Radio buttons should always be used in groups (using RadioGroup) to provide users with a clear set of mutually exclusive options. Standalone radio buttons can be confusing as they cannot be unchecked once selected.
</Note>
