## Overview

The `@shizen-ui/svelte` package provides a collection of accessible, customizable UI components built for Svelte 5. All components are built with TypeScript and leverage the `@shizen-ui/styles` package for styling.

## Installation

```bash
npm install @shizen-ui/svelte
```

## Button Component Example

The Button component demonstrates the typical structure of Shizen UI components:

```typescript
interface BaseProps extends HTMLButtonAttributes {
  variant?: ButtonVariants["variant"];
  size?: ButtonVariants["size"];
}

interface NormalProps extends BaseProps {
  iconOnly?: false;
  children?: Snippet;
  startContent?: IconContent;
  endContent?: IconContent;
}

interface IconOnlyProps extends BaseProps {
  iconOnly: true;
  children: Snippet;
  startContent?: never;
  endContent?: never;
}

type Props = NormalProps | IconOnlyProps;
```

### Key Features

- **Type-safe variants** using `tailwind-variants`
- **Svelte 5 runes** (`$props()`, `$derived()`)
- **Context integration** with ButtonGroup
- **Flexible content** with Svelte snippets
- **Icon support** via `startContent`/`endContent` or `iconOnly`

## Usage Example

```svelte
<script>
  import { Button, ButtonGroup, Input, Checkbox } from '@shizen-ui/svelte';
</script>

<ButtonGroup variant="primary" size="md">
  <Button>First</Button>
  <Button>Second</Button>
  <Button>Third</Button>
</ButtonGroup>

<Input size="md" placeholder="Enter text..." />

<Checkbox>
  <Checkbox.Control>
    <Checkbox.Indicator />
  </Checkbox.Control>
  <Checkbox.Content>Accept terms</Checkbox.Content>
</Checkbox>
```

## Component Architecture

Shizen UI components follow these patterns:

1. **Compound Components**: Complex components like `Checkbox` and `Radio` use sub-components for flexible composition
2. **Context API**: Components like `ButtonGroup` and `InputGroup` provide context to children
3. **Style Variants**: All components use `tailwind-variants` for type-safe styling
4. **HTML Attributes**: Components extend native HTML element types for full attribute support
